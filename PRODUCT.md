# anaswer.os

## Register

brand — a personal portfolio where the design IS the product. The site is a playable retro desktop OS.

## Users & Purpose

Recruiters, collaborators, and curious developers land here to get a feel for Anaswer (full-stack developer) in under a minute. The interface should reward poking around: open windows, type in the terminal, find the easter eggs.

## Personality

Cozy, playful, handcrafted. A warm 90s desktop that someone clearly loves — not a sterile template. Think worn Game Boy + a tidy desk lamp, not vaporwave neon.

## Committed identity (do not re-litigate)

The OS metaphor is the site — a 2026-07-06 experiment replacing it with a conventional dark one-pager was rejected by Anaswer; improve within the OS, don't replace it.

- **Palette**: deep teal desk (`#02656A`, sampled from the pixel-art portrait so it blends into the wallpaper), cream windows (`#F6F1E7`), ink text (`#2A2721`), amber accent (`#E8A33D`), blush + sage support.
- **Type**: Silkscreen (pixel display) / DM Sans (body) / IBM Plex Mono (terminal + metadata).
- **Imagery**: the pixel-art self-portrait (`public/portrait.png`) is the brand image — desktop hero, About avatar, favicon (`app/icon.png`).
- **Motion**: springy window physics via framer-motion; subtle portrait parallax; CRT scanline toggle. Reduced-motion respected globally.

## Anti-references

Generic SaaS landing pages, editorial-serif portfolios, glassmorphism, neon synthwave, conventional dark "developer portfolio" one-pagers.

## Accessibility

Keyboard: any-key boot, Konami code, focus-visible rings everywhere. Body text ≥ 4.5:1 on cream. Desktop text on teal stays ≥ cream/80.
